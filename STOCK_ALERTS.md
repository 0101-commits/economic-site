# 📈 투자 현황 — 가상 포트폴리오 & 카카오톡 종목 알림 가이드

대시보드의 **「투자 현황」** 페이지에서 관심 주식/ETF 를 트래킹하고,
평단가·수량으로 가상 포트폴리오를 계산하며, 조건 충족 시 **카카오톡 알림**을 받는 기능입니다.

> ⚠️ **본 정보는 15분 지연된 데이터 기준입니다.** 무료 시세 API(네이버/야후) 특성상
> 화면 시세와 카카오톡 알림 모두 최대 15분가량 지연될 수 있으며, 이 문구는 UI 에 상시 노출됩니다.

---

## 1. 기능 요약

### 투자 현황 대시보드
- **종목 리스트**: 현재가·당일 등락률을 표로 한눈에. 행 클릭 → 상세 차트.
- **종목 추가**: 한국 6자리 코드(`005930`) / 미국 티커(`AAPL`) / **한글 종목명 검색**(예: `SOL K방산`) 모두 지원.
- **그룹(폴더)**: `＋ 그룹` 버튼으로 자유롭게 생성·이름변경·삭제, 종목별 그룹 이동.
- 첫 방문 시 사용자 보유 ETF 9종(RISE 미국양자컴퓨팅, SOL 미국원자력SMR, RISE AI반도체TOP10,
  TIGER 미국S&P500, PLUS 글로벌희토류&전략자원생산기업, SOL 조선TOP3플러스,
  RISE 버크셔포트폴리오TOP10, SOL 미국AI전력인프라, SOL K방산)이 **자동 등록**됩니다.

### 가상 포트폴리오
- 종목별 **내 평단가 / 보유 수량** 입력(표 셀에서 바로 수정).
- 실시간 시세 연동 → **총 평가 금액 / 평가 손익(₩) / 수익률(%)** 자동 계산. 미국 종목은 USD/KRW 환산.
- **자산 비중 파이 차트**: 종목별 / 유형별(주식 vs ETF) 토글.
- 평단가·수량은 **브라우저(localStorage)에만 저장** — 서버로 전송되지 않습니다.

### 상세 차트 (TradingView Lightweight Charts™)
- 캔들 + 거래량, 기간 1일/1주/3개월/1년/3년.
- 보조지표 **토글(On/Off)**: 이동평균선 **MA**(5·20·60) / **RSI**(14) / **MACD**(12·26·9).
- 무료 오픈소스 라이브러리를 프론트에 이식 — 별도 서버/위젯 비용 없음.
- 신규 상장 ETF 등 야후에 없는 한국 종목은 네이버 일봉으로 자동 폴백.

### 카카오톡 알림 (조건 트리거)
| 분류 | 조건 |
|---|---|
| 가격 지정 | 가격이 목표가 **이상** / **이하** 도달 |
| 등락률 | 전일 종가 대비 **±N%** 도달 (양수=상승, 음수=하락) |
| 시장 시그널 | **52주 신고가 / 신저가** 도달 |
| 시장 시그널 | **거래량 전일 대비 N%(기본 300%) 폭등** |
| 시장 시그널 | 이동평균선 **골든크로스 / 데드크로스** (MA5/20 또는 MA20/60) |

**도배 방지(필수)** — 가격이 목표가 부근에서 횡보해도 알림이 쏟아지지 않도록, 알림마다 선택:
- **하루 1회만** (기본) — 같은 알림은 KST 기준 하루 한 번만 발송
- **1시간 쿨다운** — 발송 후 60분간 같은 알림 재발송 금지

추가 보호장치: 1회 실행당 최대 3통, 장중에만 평가(한국 09:00–15:40 / 미국 22:30–06:10 KST),
여러 알림은 한 통에 묶음 발송.

## 2. 동작 구조

```
[투자 현황 페이지] ──(☁ 서버에 저장)──▶ [Cloudflare Worker /portfolio]
                                              │ GitHub Contents API
                                              ▼
                                    alerts_config.json (저장소 커밋)
                                              │
[GitHub Actions stock-alerts.yml — 장중 15분마다]
   scripts/check_alerts.py 가 시세 조회(네이버/야후) → 조건 평가
   → 충족 시 카카오톡 발송 → alerts_state.json 에 발송 이력 커밋(도배 방지)
```

- 알림 조건만 저장소에 저장됩니다(공개 저장소 주의 — **평단가/수량은 전송 안 됨**).
- 발송 모드(나에게 보내기/친구에게 보내기)는 시황 다이제스트와 동일하게 자동 판별 — `KAKAO_SETUP.md` 참고.

## 3. 설정 (1회)

이미 시황 다이제스트(KAKAO_SETUP.md)를 설정했다면 추가로 할 일은 **Worker 재배포뿐**입니다.

1. **카카오 시크릿** — `KAKAO_REST_API_KEY`, `KAKAO_REFRESH_TOKEN`
   (GitHub Secrets, 시황 다이제스트와 공용 — 미설정 시 `KAKAO_SETUP.md` ①~④ 수행)
2. **Worker 시크릿 `GH_DISPATCH_TOKEN`** — 알림 설정을 저장소에 커밋하는 데 사용
   (kakao cron 용으로 이미 등록했다면 그대로 공용. `KAKAO_SETUP.md` 하단 참고 —
   fine-grained PAT, 이 저장소 Contents: Read and write)
3. **Worker 재배포** — `/portfolio` 엔드포인트 반영:
   ```bash
   npx wrangler deploy
   ```
4. *(필수)* **Worker 시크릿 `ALERTS_SYNC_KEY`** — 임의의 비밀 문자열(예:
   `openssl rand -hex 24`)을 등록하고, 웹 UI 의 「🔑 동기화 키」에 같은 값을 입력한
   사람만 알림 설정을 저장할 수 있습니다.

   > ⚠ **fail-closed**: 시크릿 미설정 시 저장이 503 으로 **비활성화**됩니다.
   > 과거에는 '미설정 시 무인증 쓰기 허용'이었으나, 인터넷상 누구나 알림 설정을
   > 덮어쓰고 GitHub 커밋을 유발할 수 있는 경로(OWASP API2)라 차단으로 전환했습니다.

   > 🔐 **보안**: 브라우저는 입력된 키를 **SHA-256 해시로 변환해서만 보관·전송**합니다
   > (localStorage·네트워크 어디에도 평문 키가 남지 않음). Worker 는 자기 시크릿의
   > 해시와 비교해 검증합니다. 구버전 평문 전송 호환은 제거되었습니다.
5. *(선택)* 웹 UI 의 「🔔 테스트 발송」 버튼으로 저장 직후 알림 파이프라인을 즉시
   검증할 수 있습니다 — 정규 15분 cron 을 기다리지 않고 워크플로를 1회 실행해
   "[테스트]" 프리픽스가 붙은 결과 메시지를 바로 받습니다(발송 이력은 소모하지 않음).

### 프라이버시 주의

알림 조건(종목·조건값)은 이 저장소의 `alerts_config.json` 에 커밋됩니다. **저장소가
Public 이면 어떤 종목을 어떤 가격대에 모니터링하는지 누구나 볼 수 있습니다**
(평단가·수량은 전송되지 않음). 관심 종목 노출이 부담스럽다면:

- 저장소를 **Private 으로 전환**하는 것을 권장합니다 — GitHub Pages 는 무료 플랜에선
  Public 전용이므로, Private 전환 시 호스팅을 Cloudflare Pages/Netlify 등으로 옮기거나
  GitHub Pro 를 사용해야 합니다.
- 또는 알림 기능을 쓰지 않고(서버 저장 안 함) 포트폴리오 트래킹만 사용해도 됩니다 —
  평단가·수량·종목 리스트는 브라우저 localStorage 에만 저장됩니다.

### 사용 방법
1. 「투자 현황」에서 종목 행의 **🔔** 버튼 → 조건·도배 방지 옵션 선택 → 추가
2. 상단의 **「☁ 서버에 저장」** 클릭 ← **이걸 눌러야 실제 발송에 반영됩니다**
3. 즉시 테스트: GitHub → Actions → "Stock Alerts (KakaoTalk, 장중 15분)" → **Run workflow**

### 발송 예시
```
🔔 6/11 10:15 종목 알림
SOL K방산 12,345원(+5.2%) 등락률 +5% 도달
TIGER 미국S&P500 21,840원(+0.8%) 52주 신고가
※ 15분 지연 데이터 기준
[대시보드 보기]
```

## 4. 문제 해결

- **「저장 실패: Worker 에 GH_DISPATCH_TOKEN 시크릿 필요」** → 위 ③-2 수행 후 재배포.
- **「저장 실패: 동기화 키 불일치」** → 「🔑 동기화 키」에 Worker 의 `ALERTS_SYNC_KEY` 값 입력.
- **알림이 안 옴** → ① 「☁ 서버에 저장」을 눌렀는지, ② Actions 의 stock-alerts 런 로그,
  ③ 장중인지(장외 시간엔 평가 안 함), ④ 도배 방지(하루 1회)에 걸렸는지 확인.
  발송 이력은 `alerts_state.json` 에서 볼 수 있습니다.
- **다른 기기에서 알림 설정 보기** → 「⬇ 서버에서 불러오기」 (단, 포트폴리오 평단가/수량은
  기기별 localStorage 라 동기화되지 않음).
- GitHub `schedule` 은 best-effort 라 몇 분 지연될 수 있습니다 — 알림 데이터 자체가
  15분 지연 기준이므로 분 단위 정시성은 보장하지 않습니다.
